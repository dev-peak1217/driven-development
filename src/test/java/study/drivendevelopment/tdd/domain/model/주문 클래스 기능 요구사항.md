### 구현 내용

1. 특별한 명시가 없으면 필드와 메소드는 private으로 선언한다.
2. 클래스 구현 테스트
    1. 주문 이라는 클래스의 기본 생성자 호출 테스트를 진행한다.
    2. 클래스명은 Order로 명명한다.
3. 생성자 반환 기능 테스트
    1. 해당 메소드는 정적 팩토리 메소드 패턴을 이용하여  create 이름의 메소드를 생성하고 생성자를 반환한다.
4. UUID 생성 기능 테스트
    1. java의 UUID 클래스 randomUUID 메소드를 호출하여 반환한다.
    2. 메소드의 명칭은 generateUUID로 명명한다.
5. 주문번호 생성 기능 테스트
    1. 현재날짜를 YYmmddhhmmss 형식의 문자열 값으로 반환한다.
6. 배송 주소 검증 기능 테스트
    1. 문자열 파라미터 값이 null 또는 공백을 가질 수 없는 기능을 구현한다.
    2. a번사항이 false라면 예외를 발생한다.
7. 전화번호 검증 기능 테스트
    1. 문자열 파라미터 값은 ‘3자리-4자리-4자리’ 형식을 가지며, 하이픈(-)을 제외하고 나머지는 숫자로만 구성되어야 한다.
    2. a번사항이 false라면 예외를 발생한다.
8. 결제 금액 검증 테스트
    1. 숫자 파라미터 값은 무조건 0 이상의 값이어야 한다.
    2. 소숫점의 경우는 첫번째자리에서 내림 처리를 한다.
    3. a번사항이 false라면 예외를 발생한다.
9. 필드 생성 테스트
    1. UUID 필드를 생성한다.
    2. 주문번호 필드를 생성한다.
    3. 배송 주소 필드를 생성한다.
    4. 전화번호 필드를 생성한다.
    5. 결제 금액 필드를 생성한다.
