## 1. 목적

- 도메인 계층의 주문 클래스를 생성하기

## 2. 방법

### 공통 규칙

1. 도메인 계층은 사용하는 언어 말고는 어떤 라이브러리와 프레임워크에 의존성을 가지면 안된다.
2. 예를 들어 java의 경우라면 POJO로 작성한다.
3. 각 메소드의 이름은 의도를 가지도록 작성한다.
    1. ex) UUID는 generateUUID(), generateOrderNumber()

### 구현 내용

1. class 이름은 Order로 생성한다.
2. 필드 생성
    1. UUID, 주문번호, 주문 수량, 배송주소, 핸드폰 번호, 결제 금액을 생성한다.
3. 생성자 생성
    1. 적정 팩토리 메소드 패턴을 이용하여 create 메소드를 생성
    2. UUID와 주문번호는 Order 클래스의 내부 메소드에서 자체 생성
    3. 그 외 필드 값은 파라미터로 전달된 아규먼트 값을 사용
4. 메소드 생성
    1. UUID class를 통한 randomUUID 생성
    2. 주문 번호를 생성하는 메소드
        1. 현재 시간을 YYmmddhhmmss 으로 표현하여 생성
    3. 배송주소 검증 메소드는 null 또는 공백은 불가하다.
    4. 전화번호 검증 메소드는 3자리-4자리-4자리 숫자만으로 가능하다.
    5. 결제금액 검증 메소드는 0 또는 1 이상의 값만 된다.
        1. 음수는 허용하지 않는다.
        2. 소수점의 경우 절삭한다.
